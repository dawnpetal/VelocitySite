Tutorial 02

Workspaces

Organise scripts into folders, open multiple tabs, use the command palette, and set up autoexecute for scripts that run on every inject.

01

Organise scripts into folders

Use the buttons provided at the top in the file explorer to create a new folder or file. A sensible starting layout:

Workspace 
games/
  obby-skip.lua
  speed-hub.lua
utils/
  anti-afk.lua
  fps-unlocker.lua
If you want the correct file icon and syntax, please suffix .lua at the end the file name.
02

Open multiple tabs

Click any file in the sidebar to open it in the editor. Click more files to stack them as tabs across the top. Middle-click or hit the × on a tab to close it. Unsaved tabs show a dot indicator.

03

Use the command palette

Press Cmd + P to fuzzy-search any open file by name. Press Cmd + Shift + P for editor commands, toggle word wrap, format document, change language mode, and more.

04

Using autoexecute

Open the autoexecute tab in the sidebar and create scripts inside it. Everything in there runs automatically everytime you join a game from that point forward. Right place for anti-AFK, UI patches, anything you want active every session.

autoexecute/init.lua lua 
local RunService = game:GetService("RunService")
local VIM = game:GetService("VirtualInputManager")

local t = 0
RunService.Heartbeat:Connect(function(dt)
  t = t + dt
  if t >= 900 then
    VIM:SendKeyEvent(true, "Space", false, nil)
    t = 0
  end
end)
Scripts run in filesystem order. Prefix filenames with numbers (01-init.lua, 02-ui.lua) to control the sequence.
